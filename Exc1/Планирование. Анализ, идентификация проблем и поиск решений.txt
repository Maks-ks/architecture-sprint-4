1. Существующие и потенциальные проблемные места
    - увеличилось время выполнения заказов, хотя производственные мощности могут обеспечить двукратный рост
    - долго прогружается начальная страница с заказами в MES 
    - время расчета стоимости изделия в MES может достигать 30 минут
    - в текущем варианте MES может стать точкой отказа всей системы, если открыть ее API для всех пользователей
    - увеличивается время релизного цикла, если ничего не предпринять, можно отстать от конкурентов и потерять клиентов

2. Инициативы по устранению проблем
    - нанять дополнительных специалистов для реализации плана по устранению проблем
    - сделать систему более наблюдаемой (внедрив мониторинг, логирование, трейсинг), чтобы более точно выявить источники проблем с производительностью и в реальном времени понимать, что с системой происходит
    - разбить MES на микросервисы (начать с расчета стоимости изделия и заказов), чтобы проще масштабироваться при увеличении нагрузки на API
    - внедрить кеширование в MES
    - оптимизировать алгоритм расчета стоимости изделия, чтобы сократить время

3. Приоритеты
    Для решения существующих проблем сперва нужно:
    - усилить команду, так как текущий состав не справится за пол года с нашим амбициозным планом.
        Нам дополнительно потребуется:
        + 3 C# разработчика для разбиения MES на микросервисы и поддержки полученного решения. MES написана на C#, поэтому микросервисы планируется реализовывать на этом же стеке, для ускорения процесса.
        + 2 QA инженера для автоматизации процесса тестирования. Это позволит минимизировать человеческий фактор, ускорить процесс тестирования и выявлять критические ошибки на ранних стадиях, что в свою очередь сократит время релизного цикла. 
        + 2 DevOps-инженера для внедрения observability системы и донастройки CI/CD. Это позволит мониторить систему в моменте, находить узкие места, а также делать бизнес-гипотезы по поведенческому фактору пользователей.
        + 2 SRE инженера для мониторинга систем и управления инцидентами.
    - внедрить в систему мониторинг, логирование, трейсинг (этот пункт в идеале нужно делать параллельно с предыдущим)
    - разбить MES на микросервисы в целях масштабирования
    - внедрить кэширование в MES
    - оптимизировать алгоритм расчета стоимости изделия, чтобы сократить время

4. Целевая архитектура через полгода
    Через полгода команда внедрит в систему мониторинг, логирование, трейсинг. MES будет разбита на независимые микросервисы для лучшего масштабирования и оптимизации расчета стоимости изделий.
Будет внедрено кэширование для повышения отзывчивости системы и снижения нагрузки. Стабилизируется время выпуска новых релизов.

5. Если говорить только о трех пунктах, которые можно выбрать, то я бы выбрал:
    - нанять дополнительных специалистов для реализации плана по устранению проблем
    - сделать систему более наблюдаемой (внедрив мониторинг, логирование, трейсинг)
    - разбить MES на микросервисы
так как выполнение этих пунктов позволит выявить точные проблемные места системы и оперативно их исправить, отслеживать ее состояние в реальном времени, 
справится с увеличением заказов, не потерять клиентов (и выручку как следствие). Проблемы внутренних пользователей можно решать параллельно основным, при наличии времени.
